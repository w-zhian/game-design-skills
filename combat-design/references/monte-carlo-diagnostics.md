# Monte Carlo 战斗公式诊断框架

> 来源：combat-formula-analyzer skill
> 用途：通过大规模Monte Carlo模拟（10000+次）对战斗公式进行多维度统计诊断

## 模拟规模标准

| 测试场景 | 模拟次数 | 说明 |
|---|---|---|
| 基础伤害测试 | 10,000 | 固定攻/防/暴击，观察伤害分布 |
| 防御收益曲线 | 50次/档 × 20档 = 1,000 | 防御0~1000，每50一档 |
| 暴击率验证 | 10,000/档 × 11档 | 暴击率0%~100%，每10%一档 |
| 技能倍率扫描 | 10次/倍率 × 30种 = 300 | 倍率0.5x~15x |
| **总测试次数** | **≥ 21,300** | |

## 六维诊断体系

### 1. 防御收益诊断（Defense Diminishing Returns）
- **正常**：防御越高，每点防御提供的减伤越少（递减）
- **异常**：
  - 防御收益不递减 → 🔴 缺少软上限，防御可无脑堆
  - 防御收益递减过快（K值过小）→ 🟡 后期防御完全无用
  - 防御收益为0或负数 → 🔴 公式写反了
- **检验**：`dmg_reduction[def=100] vs dmg_reduction[def=200]`，200的减伤应>100但增幅<100的增幅

### 2. 暴击系统诊断（Crit Distribution）
- **正常**：10000次模拟，暴击率10%应触发约1000次（±3%误差）
- **异常**：
  - 实际暴击率偏离预期>5% → 🔴 随机数生成器问题
  - 暴击伤害无波动 → 🟡 缺少暴击乘区
  - 暴击永不触发 → 🔴 随机数范围写反
- **检验**：`|实际 - 期望| / 期望 < 5%`

### 3. 伤害方差诊断（Damage Variance）
- **正常**：伤害浮动±10%~±20%，有手感但不太看脸
- **异常**：
  - 方差=0 → 🟠 无随机性，战斗无聊
  - 方差>均值×50% → 🟡 随机性过大
  - 存在极端值（>均值×10或<均值×0.1）→ 🟡 浮动范围过大
- **检验**：`CV = std_dev / mean`，合理范围CV ∈ [0.05, 0.30]

### 4. 乘区隔离诊断（Multiplicative Region）
- **正常**：各乘区独立叠乘，总乘区有上限
- **异常**：
  - 总乘区无上限 → 🔴 数值无限膨胀
  - 某乘区永远最优 → 🟠 乘区失衡，Build单一
  - 某乘区永远无用 → 🟡 乘区设计失效
- **检验**：检查各乘区是否有超限判断（min/max）

### 5. 数值溢出诊断（Overflow）
- 单次伤害>1,000,000 → 🟠 可能溢出
- 伤害<0 → 🔴 公式错误（治疗而非伤害）
- NaN/inf → 🔴 除零或无效运算

### 6. 公式单调性诊断（Monotonicity）
- **正常**：攻击↑→伤害↑，技能倍率↑→伤害↑
- **异常**：攻击提升但伤害不增或下降 → 🔴 公式逻辑错误

## 输出报告标准结构

| Sheet | 内容 |
|-------|------|
| Sheet1 | 执行摘要（问题数量/严重程度分布） |
| Sheet2 | 防御收益曲线（def 0~1000，对应减伤率） |
| Sheet3 | 暴击率验证（11个档位，期望 vs 实际） |
| Sheet4 | 伤害分布（直方图数据，均值/中位数/方差/CV） |
| Sheet5 | 乘区诊断（各乘区贡献度） |
| Sheet6 | 极端值清单（异常伤害记录） |
| Sheet7 | 详细日志（10000条原始伤害数据） |
