# System Design Doc Template

Use when the user needs a fuller design document.

## Summary

One paragraph explaining the system and why it exists.

## Goals And Non-Goals

List what the system should achieve and what it should not solve.

## Player Experience

Describe the player journey through the system from discovery to repeated use.

## System Model

- Entities.
- States.
- Player verbs.
- Resources.
- Rules.
- Rewards.
- Limits.
- Failure cases.

## Loop Integration

Explain how the system connects to core loop, meta loop, economy, content, onboarding, and live ops.

## Progression

Describe unlocks, pacing, mastery curve, and lifecycle stage differences.

## Economy And Rewards

List sources, sinks, caps, exchange paths, and risk points.

## UX Requirements

List screens, feedback, player-readable states, and tutorial needs.

## Balance And Tuning

List knobs, default assumptions, test ranges, and side effects.

## Risks

False choices, dominant strategies, player confusion, production cost, content exhaustion, monetization pressure.

## Validation Plan

Playtest tasks, telemetry, expected behaviors, and decisions to make after data.
