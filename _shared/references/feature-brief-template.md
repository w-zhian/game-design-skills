# Feature Brief Template

Use when the user needs a concise feature proposal.

## Feature

Name and one-sentence purpose.

## Player Problem

What player need, frustration, aspiration, or behavior this feature addresses.

## Design Goal

The intended player behavior or system effect.

## Core Loop Fit

How the feature touches action, feedback, reward, progression, and return.

## Rules

- Entry condition.
- Player action.
- System response.
- Reward or consequence.
- Limits and cooldowns.
- Failure or edge cases.

## Tuning Knobs

Name each adjustable value, expected effect, and likely side effect.

## Risks

Dominant strategy, economy inflation, UI overload, onboarding burden, content cost, monetization trust risk.

## Validation

What to test, with whom, and what observation or metric would confirm success.
