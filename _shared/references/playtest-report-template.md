# Playtest Report Template

Use when converting playtest notes into design decisions.

## Build And Test Context

Version, target feature, player segment, session length, and test goal.

## Intended Behavior

What designers expected players to do and feel.

## Observed Behavior

Separate direct observations, player quotes, and telemetry.

## Diagnosis

Name the likely root cause: clarity, incentive, difficulty, economy, pacing, UX, content, or social pressure.

## Segment Impact

Who is affected: new player, average player, expert, payer, non-payer, competitive, collector, social player.

## Recommended Changes

Start with the smallest change. For each change, list expected effect and side effect.

## Follow-Up Test

What to test next, what success looks like, and what decision the result will unlock.
