# 跨品类数值分析框架

> 来源：cross-genre-numeric-designer-perspective skill
> 用途：跨MMO/RPG/SLG/卡牌/Roguelike品类的数值设计方法论

## 六维数值分析法

### 维度1：战斗数值（Combat）
**伤害公式决策树**：
```
选择伤害公式：
├─ 简单直观优先？ → 减法公式（攻击-防御）⚠️需保底防不破防
├─ 策略博弈优先？ → 乘法公式（攻击×(1-减伤率)）✅防御效果百分比化
└─ 长线运营优先？ → 除法公式（攻击²/(攻击+防御)）★推荐 ✅攻防均衡
```

**属性权重换算**（以1攻击为基准）：
| 属性 | 权重 | 说明 |
|------|------|------|
| 攻击 | 1.0 | 基准 |
| 防御 | 0.7 | 按战斗时长换算 |
| 生命 | 0.2 | 按战斗时长换算 |
| 暴击 | 1.5 | 考虑暴击率×暴击伤害期望 |
| 攻速 | 1.2 | 根据技能CD结构 |

### 维度2：成长数值（Progression）
**经验曲线选择**：
| 曲线类型 | 公式 | 适用场景 | 代表游戏 |
|----------|------|----------|----------|
| 线性 | 经验=等级×常数 | 简单直接 | 早期RPG |
| 多项式 | 经验=等级^1.5×常数 | 中等复杂度 | 一般手游 |
| 指数 | 经验=常数×(1.2^等级) | 延长周期 | MMO |
| 分段 | 不同阶段不同曲线 | 精细化控制 | 原神/艾尔登法环 |

**成长感知标准**：
- 每级提升：至少5%战力增长
- 装备更新：新装备比旧装备强20%+
- 技能解锁：每5级有新技能或强化

### 维度3：经济数值（Economy）
**经济闭环三角**：
```
      资源产出
       / \
      /   \
  资源消耗 — 资源转化
```
- 产出（Faucet）：击杀、任务、活动、签到
- 消耗（Drain）：升级、强化、抽卡、商店
- 转化：资源之间的兑换（金币→钻石）

### 维度4：平衡数值（Balance）
**PvE平衡**：DPS=敌人HP/目标击杀时间
**PvP平衡**：同战力胜率50%±5%，避免绝对克制
**付费平衡**：免费/付费差距控制在时间维度（3-7天）

### 维度5：商业化数值（Monetization）
| 付费层级 | 月均消费 | 设计目标 |
|---------|---------|---------|
| 免费玩家 | ¥0 | 提供完整体验，贡献DAU和社交 |
| 小R | ¥30-100 | 月卡+通行证，体验加速 |
| 中R | ¥100-500 | 礼包+月卡，战力领先1-2周 |
| 大R | ¥500+ | 追求排名和收藏，撑营收 |

### 维度6：留存数值（Retention）
- D1留存目标：40%+
- D7留存目标：20%+
- D30留存目标：10%+
- 卡点设计：每个卡点对应一个付费转化点

## 数值安全边界标准

| 品类 | 付费/免费差距 | 版本战力增长 | 同版本角色差距 |
|------|-------------|-------------|---------------|
| MMO | ≤3倍 | ≤2倍/版本 | ≤1.5倍 |
| 手游RPG | ≤5倍 | ≤3倍/版本 | ≤2倍 |
| SLG | ≤10倍 | ≤5倍/赛季 | ≤3倍 |
| 卡牌 | ≤5倍 | ≤3倍/版本 | ≤2倍 |
