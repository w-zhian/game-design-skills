# 幸存者/割草类数值框架

> 来源：vampire-survivors-numeric-perspective skill
> 用途：类吸血鬼幸存者游戏的精确数值公式与平衡方法论
> 参照产品：Vampire Survivors/弹壳特攻队/土豆兄弟

## 核心心智模型

### N1：三层DPS体系
```
总DPS = 武器层DPS × 属性乘区 × 协同乘区

武器层DPS = Σ(武器基础伤害 / 冷却时间) × 命中率 × 武器修正
属性乘区 = (1 + ATK%) × (1 + 暴击率% × 暴伤%) × (1 + 攻速% / CD)
协同乘区 = Π(1 + C_i) = (1+C₁) × (1+C₂) × ... × (1+Cₙ)
```

### N2：涌现约束下的平衡
```
设计涌现空间，而非穷举Build组合。
最优Build DPS / 最差Build DPS ≤ 5倍 → 平衡可接受
```

### N3：数字膨胀的边界
```
感知上限 = 10^9（10亿）
超出上限 → 99,999,999,999 ≈ 10^11 ≈ 无感

推荐：
- 最终DPS ≤ 10^8
- 总协同乘区 ≤ ×5.0
- 最强Build / 平均Build ≤ 3倍
```

### N4：内容即敌人
```
幸存者-like：敌人数值设计 = 90%的内容设计
→ 波次设计 = 节奏控制
```

## 武器DPS基准计算

### 五种武器类型DPS修正

| 类型 | 覆盖系数 | 说明 |
|------|---------|------|
| A类(范围持续) | 1.0 | 全程覆盖，最稳定 |
| B类(投射追踪) | HitRate × 0.9 | 追踪损失 |
| C类(地面放置) | Duration / Cooldown | 时间覆盖比 |
| D类(召唤) | 召唤数量 × 0.8 | 多体叠加 |
| E类(环境强化) | 1.0 | 不直接伤害 |

### DPS计算公式
```
DPS_Actual = DPS_Base × ProjectileCount × (1 + PierceCount × 0.3) × 覆盖系数
DPS_Base = BaseDamage / Cooldown × HitRate
```

## 敌人HP缩放

```
HP_Actual = HP_Base × ScalingFactor^(WaveID-1)

推荐缩放系数：
- 主游戏：1.12~1.15
- 无尽模式：1.20~1.25
```

**波次1参考基准**：
| 敌人类型 | HP | 目标击杀时间 | 所需玩家DPS |
|---------|-----|------------|------------|
| 普通 | 30 | 2-5s | 6-15 |
| 精英 | 150 | 8-15s | 10-19 |
| BOSS | 2000 | 30-60s | 33-67 |

## 平衡检查标准

| 指标 | 合理范围 | 异常信号 |
|------|---------|---------|
| 最优/最差Build DPS比 | ≤5倍 | >5倍→Build严重失衡 |
| 总协同乘区 | ≤×5.0 | >5.0→数值膨胀 |
| 最终DPS | ≤10^8 | 超出感知上限 |
| 每10级战力增幅 | 50%~80% | <50%成长不足，>80%膨胀 |

## 付费设计三红线（VS类）

1. **不卖Build解**：付费不能获得免费玩家无法获取的武器/技能
2. **不卖数值乘区**：付费获得的属性加成不进入独立乘区（只能加入已有乘区）
3. **不卖时间压缩**：付费不能跳过波次/加速刷怪（割草游戏的节奏即体验）
