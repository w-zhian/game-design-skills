# 数值公式工具库

> 来源：game-numeric-design skill（《游戏数值设定入门》方法论）
> 用途：数值设计的基础公式工具与流程参考

## 五特性模型

任何数值模块都具备五个特性：

| 特性 | 说明 | 示例 |
|------|------|------|
| 生成 | 模块产生的原因和过程 | 怪物生成、装备掉落 |
| 成长 | 从起始到成型的轨迹 | 等级提升、属性成长 |
| 消亡 | 消失和灭亡 | 装备淘汰、技能失效 |
| 变化 | 不单向成长，受环境影响 | 属性波动、爆率变化 |
| 联系 | 模块之间的关系 | 装备影响战斗力 |

## 数值设定流程

```
1. 基本功能假设 → 这个模块解决什么问题
2. 建立数据模型 → 用什么数学结构表达
3. 设定数值方向 → 希望玩家感受到什么
4. 调整数值实现方向 → 具体参数微调
5. 验证并输出 → 模拟测试+配表
```

## 公式工具库

### 1. 加减乘除（线性公式）
适用于：线性变化、直观数值
```javascript
attack = baseAttack + strength * 1        // 力量影响攻击
expToLevel = baseExp + level * 1000       // 等差升级经验
damage = attack - defense                 // 减法伤害
```

### 2. 幂函数
适用于：成长曲线、边际效应
```javascript
exp = 1000 * Math.pow(level, 0.667)       // 升级经验：先易后难 (i<1)
comboScore = base * Math.pow(2, comboCount) // COMBO得分：指数爆炸 (i>1)
attr = baseMax / (max - current)          // 经验衰减 (i<0)
```

### 3. 数组/数列
适用于：固定数值表、阶段成长
```javascript
pointPerLevel = 3 + Math.floor(level / 5)  // 等差数列：每5级多获得1点
gold = baseGold * Math.pow(1.1, level)     // 等比数列：金币产出
```

## 公式选择决策

| 需求 | 推荐公式 | 典型场景 |
|------|---------|---------|
| 简单直观 | 减法（攻击-防御） | 早期RPG、休闲游戏 |
| 策略博弈 | 乘法（攻击×(1-减伤率)） | 卡牌、策略 |
| 长线运营 | 除法（攻击²/(攻击+防御)） | MMO、长线手游 ★推荐 |
| 多乘区叠加 | 混合法 | 原神、崩坏等二次元 |

## 配表生成标准

### 三行表头规范
```
第1行：英文字段名（WeaponID, BaseDamage, ...）
第2行：中文字段名（武器ID, 基础伤害, ...）
第3行：数据类型（string, float, int, ...）
第4行起：数据
```

### 命名规范
- 主键：`{表名}ID`，全大写驼峰
- 外键：`{引用表名}ID`
- 布尔值：`Is{Xxx}`（IsLocked, IsActive）
- 枚举值：全大写+下划线（WEAPON_TYPE_A）
