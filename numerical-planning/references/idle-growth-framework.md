# 增量/放置游戏成长框架

> 来源：idle-game-numeric-designer-perspective skill
> 用途：增量/放置游戏的成长曲线、离线收益、Prestige设计方法论
> 参照产品：Cookie Clicker/Adventure Capitalist/Clicker Heroes/AFK Arena/放置奇兵/少年三国志/咸鱼之王

## 两条设计血脉

### 血脉A：海外纯Idle（Cookie Clicker系）
```
核心循环：等待→获取资源→购买升级→自动产出增加→循环
数学本质：递归增长——产出的增长使产出增长更快
设计灵魂：让数字讲故事（10K→1M→1B→1T本身就是叙事）
付费模式：买断/广告/无付费
```

### 血脉B：国内放置手游（AFK Arena系）
```
核心循环：挂机积累→推图验证→抽卡养成→更强→推更远
数学本质：异步养成+数值验证——放置是留存工具，不是核心玩法
设计灵魂：时间即货币——付费买时间，而非买结果
付费模式：F2P+Gacha+VIP
```

## 四大核心原则

1. **成本增长 > 收益增长** — 增量游戏存在的数学基础，违反即崩溃
2. **离线收益是双刃剑** — 足够让玩家安心离开，但不至于替代在线动力
3. **Prestige是"新游戏+"而非惩罚** — 每次重置应带来质变，而非仅是数字缩放
4. **付费差距可存在但必须可追赶** — 允许差距，但追赶路径必须清晰

## 关键差异矩阵

| 维度 | 海外Idle | 国内放置 |
|------|---------|---------|
| 成长公式 | Cost=base×rate^n（纯指数） | Cost=base×n^exp×(1+mod)（混合） |
| 离线效率 | 0%~100%（主流50%~100%） | 80%~100% |
| 时间上限 | 无~72h | 8~24h |
| Prestige | 核心机制 | 无（用转生替代） |
| 付费 | 买断/广告 | F2P+Gacha+VIP |
| 数值膨胀 | 接受 | 严格控制 |
| 社交 | 几乎无 | 公会/竞技场/跨服 |
| 终局 | 数字无限增长 | PVP排名/收集完成度 |

## 离线收益四种模型

| 模型 | 公式 | 特点 | 适用场景 |
|------|------|------|---------|
| 线性 | 收益=在线DPS×时间×ε | 简单直观 | 放置手游主流 |
| 对数 | 收益=在线DPS×log(时间)×ε | 收益递减 | 防止长时间挂机 |
| 硬上限 | 收益=在线DPS×min(时间,上限)×ε | 简单粗暴 | 国内放置（8-24h上限） |
| 软上限 | 收益=在线DPS×(1-e^(-时间/τ))×ε | 平滑递减 | 追求体验平滑 |

ε = 离线效率系数（0.5表示离线只有在线一半收益）

## Prestige设计要点

**海外Idle Prestige**：
- 每次Prestige重置所有进度，获得永久乘数
- 核心乐趣：每次重来都比上次更快到达更高的高度
- 数学本质：递归增长×乘数加速

**国内放置"转生"替代**：
- 不完全重置（保留角色等级/装备）
- 重置的是关卡进度/日常资源
- 核心乐趣：每次"转生"解锁新系统/新养成线

**设计陷阱**：
- Prestige惩罚过重 → 玩家不愿重置 → 游戏失去核心循环
- Prestige奖励过低 → 重置没有意义 → 玩家感觉白干
- Prestige层级过多 → 新玩家被吓退 → 入门门槛过高
