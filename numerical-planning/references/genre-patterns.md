## 1. Cross-Genre Reading Rules

Never judge a system outside its genre promise.

| Genre promise | Numerical priority |
|---|---|
| Skill expression | Formula clarity, counterplay, low hidden power gap, readable breakpoints. |
| Collection aspiration | Long-term targets, controlled rarity, pity, duplicate value, roster breadth. |
| Strategic planning | Resource conversion, long-term opportunity cost, alliance/social pressure. |
| Relaxed habit | Low cognitive burden, smooth reward cadence, non-punitive loss. |
| Competitive legitimacy | Matchmaking, compression, anti-snowball, counter options. |
| Live-service retention | Tunable economy, event cadence, catch-up, content pacing. |

## 2. RPG And MMO

Core skeleton:

- Character level, gear level, skill/talent tree, dungeon difficulty, drops, crafting, economy.
- Power often combines base stats, gear score, skill coefficients, buffs, mitigation, and set bonuses.

Audit focus:

- Whether gear score hides or reveals real combat power.
- Whether dungeon difficulty grows with player mastery or only stat requirements.
- Whether roles have numerical value: tanking, healing, control, support, mobility.
- Whether crafting and drops compete or complement each other.
- Whether catch-up invalidates veteran progression.

Common strengths:

- Many parallel goals: gear, skill, collection, social status, mastery.
- Strong long-term aspiration through rare drops and raids.
- Role diversity can create deep social gameplay.

Common weaknesses:

- Gear treadmill erases old content too fast.
- Set bonuses or multipliers create mandatory builds.
- Late-game resource inflation makes new content hard to price.
- Raid stat checks can become exclusionary.

## 3. Card, Collection, And Gacha RPG

Core skeleton:

- Roster acquisition, rarity, duplicates, ascension, equipment, stamina, stages, events, PvP.
- Power is often unit base strength plus synergy, rarity, duplicate depth, gear, and team composition.

Audit focus:

- Expected time or spend to reach target roster depth.
- Pity threshold vs free currency earn rate.
- Duplicate conversion fairness.
- Whether low-rarity units retain strategic value.
- Whether new banners create power creep or horizontal options.
- Whether stamina gates protect pacing or block engagement.

Common strengths:

- Clear collection aspiration.
- Team synergy creates combinatorial depth.
- Events can refresh goals without rebuilding the core game.

Common weaknesses:

- Power creep invalidates old spending.
- PvP exposes payer gap too directly.
- Too many growth materials create confusing bottlenecks.
- Duplicate systems can feel punitive if conversion value is poor.

## 4. SLG, 4X, And War Strategy

Core skeleton:

- Base construction, resource production, research, troops, march time, alliances, territory, PvP loss.

Audit focus:

- Production vs consumption balance by city level.
- Time acceleration economy and paid speedups.
- Troop loss severity and recovery time.
- Whale pressure, alliance protection, and server lifecycle.
- Research tree marginal value and bottleneck pacing.
- New-server and merge-server catch-up mechanics.

Common strengths:

- Deep long-term planning and social dependency.
- Resource tradeoffs create meaningful opportunity cost.
- Alliance goals can retain players beyond individual progression.

Common weaknesses:

- Strong players can permanently suppress weak players.
- Paid acceleration can create irreversible server hierarchy.
- Timers replace decisions if too many systems run in parallel.
- Cross-server events can shock local economy and morale.

## 5. Idle And Incremental

Core skeleton:

- Automated production, upgrade multipliers, offline income, prestige/reset, ad or IAP boosts.

Audit focus:

- Multiplier stacking and runaway growth.
- Prestige timing: when reset becomes attractive.
- Offline income cap and return frequency.
- Ad reward expected value.
- Whether decisions matter or upgrades are always obvious.

Common strengths:

- Clear numerical feedback and low-friction habit loop.
- Exponential growth creates strong early satisfaction.
- Prestige can refresh progression repeatedly.

Common weaknesses:

- Growth can become meaningless if every number inflates together.
- Optimal play may collapse into watching ads.
- Reset systems can feel punitive if benefits are unclear.
- Too few decision layers turns the game into a waiting room.

## 6. Action, ARPG, And Fighting

Core skeleton:

- Player skill, frame timing, enemy patterns, damage, defense, cooldowns, stamina/energy, equipment.

Audit focus:

- Ratio of player skill to numerical power.
- Whether damage windows and defense windows are readable.
- Cooldown, resource, and animation-lock fairness.
- Build diversity without erasing mechanical mastery.
- Difficulty curve: new mechanics vs bigger numbers.

Common strengths:

- High agency and mastery expression.
- Numerical progression can broaden playstyle.
- Moment-to-moment feedback is strong.

Common weaknesses:

- Stat power can erase the need for skill.
- One defensive stat can trivialize encounter design.
- Enemies become sponges if difficulty relies on HP alone.
- Cooldown math can create uninteractive burst cycles.

## 7. Shooter And Tactical PvP

Core skeleton:

- Weapon stats, recoil, time-to-kill, armor, movement, abilities, matchmaking, ranking.

Audit focus:

- Time-to-kill distribution.
- Headshot/bodyshot ratio and weapon role.
- Economy of armor, ammo, consumables, or abilities.
- Matchmaking fairness and rank inflation.
- Paid cosmetics vs gameplay-affecting power.

Common strengths:

- Clean skill expression through aim, positioning, and timing.
- Small numerical changes create meaningful weapon identity.
- Competitive legitimacy can retain players for years.

Common weaknesses:

- Hidden aim assist, recoil, or hitbox changes erode trust.
- Low TTK reduces counterplay; high TTK can reduce lethality fantasy.
- One weapon stat profile can dominate all ranges.
- Progression unlocks can become unfair in PvP.

## 8. MOBA And Hero PvP

Core skeleton:

- Heroes, roles, lanes, economy, item builds, cooldowns, scaling, objectives, matchmaking.

Audit focus:

- Early, mid, late game power curves by role.
- Gold/XP snowball and comeback mechanics.
- Item efficiency and build diversity.
- Hero counter relationships.
- Objective reward vs risk.
- Match length and surrender/churn points.

Common strengths:

- Strong strategic diversity from role and team composition.
- Patchable live balance through hero coefficients and item stats.
- Skill and macro decisions both matter.

Common weaknesses:

- Snowball can remove agency too early.
- Item efficiency can solve builds into one path.
- High mobility or control invalidates entire hero classes.
- Matchmaking quality dominates perception of balance.

## 9. Roguelike And Run-Based Games

Core skeleton:

- Run economy, random rewards, relics, build synergies, escalating difficulty, meta unlocks.

Audit focus:

- Randomness vs controllability.
- Synergy density and dead choices.
- Difficulty scaling within a run.
- Meta-progression effect on skill legitimacy.
- Recovery paths after bad luck.

Common strengths:

- High replayability from combinatorial variance.
- Strong emotional peaks from build breakthroughs.
- Failure can still feel fair if learning is clear.

Common weaknesses:

- Too many dead rewards make runs feel predetermined.
- Meta progression can overpower run skill.
- Random spikes can kill players without teachable counterplay.
- Infinite scaling often exposes formula flaws.

## 10. Casual, Puzzle, And Merge

Core skeleton:

- Level difficulty, move economy, boosters, lives/energy, streaks, events, collection layers.

Audit focus:

- Fail rate by level band.
- Booster value and whether boosters teach bad habits.
- Lives/timer pressure.
- Difficulty spike placement.
- Event layering and cognitive load.
- Ad/IAP relief timing.

Common strengths:

- Simple goals and fast feedback.
- Difficulty can be tuned level by level.
- Boosters give clear monetization hooks.

Common weaknesses:

- Hard levels can feel rigged if randomness dominates.
- Too many event layers overwhelm casual players.
- Booster economy can become mandatory rather than helpful.
- Repeated failure before trust is built causes churn.

## 11. Simulation And Management

Core skeleton:

- Production chains, demand, staffing, upgrades, expansion, pricing, logistics, prestige.

Audit focus:

- Production bottleneck readability.
- Chain depth vs cognitive load.
- Marginal ROI of upgrades.
- Automation timing.
- Expansion cost vs income growth.
- Whether optimal play has interesting tradeoffs.

Common strengths:

- Satisfying optimization and visible system causality.
- Multiple bottlenecks create planning depth.
- Automation can create long-term progression.

Common weaknesses:

- One bottleneck can dominate all decisions.
- Production chains become opaque.
- Upgrade ROI may become mathematically obvious.
- Waiting replaces planning if automation arrives too late.

## 12. Sports And Racing

Core skeleton:

- Player/car stats, control skill, upgrade parts, matchmaking, event tiers, stamina/fuel, ranked ladders.

Audit focus:

- Difference between skill, stats, and randomness.
- Upgrade tier breakpoints.
- Matchmaking by rating vs power.
- Event entry requirements.
- Pay advantage in competitive modes.

Common strengths:

- Clear fantasy and measurable improvement.
- Skill expression can coexist with progression.
- Equipment tuning can create identity.

Common weaknesses:

- Stat upgrades can make races feel predetermined.
- Matchmaking that mixes power and skill poorly feels unfair.
- Upgrade parts fragment into too many bottlenecks.
- Competitive monetization can damage legitimacy.
