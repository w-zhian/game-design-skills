# Content Framework

Use this reference when designing level structure, mission design, encounter pacing, and content cadence.

## 1. Content Recipe

Every piece of authored content should follow this beat structure:

| Beat | Purpose | Player Experience |
|------|---------|-------------------|
| Hook | Show a clear promise or threat | "Oh, this looks interesting/dangerous" |
| Teach | Let the player understand the main rule safely | "I see how this works" |
| Test | Ask the player to use the rule under pressure | "I need to execute this correctly" |
| Twist | Combine the rule with a previous rule or constraint | "Oh, now it's different" |
| Payoff | Reward mastery, reveal content, or unlock a new choice | "That was worth it" |

If a content piece skips any beat, it's incomplete.

## 2. Variation Axes

When designing multiple content pieces, vary ONE major axis at a time:

| Axis | Example | New Decision Created |
|------|---------|---------------------|
| Enemy mix | Add ranged enemies to melee-only | "Do I close distance or take cover?" |
| Terrain | Add verticality | "Do I hold high ground?" |
| Objective | Change from "kill all" to "survive timer" | "Do I fight or evade?" |
| Time pressure | Add countdown | "Do I optimize for speed?" |
| Resource scarcity | Limit ammo/mana | "Do I conserve or commit?" |
| Rule modifier | Add "no healing" rule | "Do I play safe or aggressive?" |

Varying visuals without varying decisions = content treadmill.

## 3. Difficulty Curve Design

### Curve Phases
| Phase | Player Stage | Difficulty Growth | Content Focus |
|-------|-------------|-------------------|---------------|
| Learning | Tutorial | Flat or gentle | Teach mechanics |
| Growth | Early game | Linear increase | Introduce variety |
| Mastery | Mid game | Non-linear (some spikes) | Combine mechanics |
| Expression | Late game | Plateau with optional challenges | Player choice |
| Endless | Endgame | Infinite scaling (if applicable) | Build optimization |

### Difficulty Spike Rules
- Spikes should be readable: the player should understand WHY it's harder
- Spikes should be preceded by a teaching moment
- After a spike, provide a brief relief (easier content to reinforce mastery)
- Never have two consecutive spikes without a breather

### Catch-Up Design
- Late joiners should reach "current content" within a reasonable time
- Catch-up should not invalidate veteran effort (don't give away what was earned)
- Catch-up mechanisms: XP boost, reduced costs, shared progression, retroactive rewards

## 4. Content Cadence Planning

### Pacing Template
| Day | Content Focus | New Mechanics | Difficulty | Reward Tier |
|-----|--------------|---------------|------------|-------------|
| Day 1 | Core loop tutorial | Basic verb | Easy | Immediate |
| Day 2-3 | Loop expansion | First upgrade | Gentle | Near-term |
| Day 4-7 | First content arc | Variety introduction | Moderate | Weekly |
| Day 8-14 | Mid-game content | Mechanic combination | Spikes | Mid-term |
| Day 15-30 | Long-term content | Mastery expression | High | Long-term |
| Day 30+ | Endgame + events | Meta challenges | Variable | Collection |

### Event Cadence
- **Frequency**: 1 event every 2 weeks (minimum)
- **Duration**: 7-14 days
- **Type rotation**: Combat event → Collection event → Social event → Story event
- **Reward integration**: Event rewards should supplement, not replace, permanent progression
- **FOMO control**: Event-exclusive items should be cosmetic or breadth, not exclusive power

## 5. Content Reuse Audit

For each piece of reused content, answer:

| Question | Acceptable | Unacceptable |
|----------|-----------|--------------|
| What changes? | Player decisions | Only visuals |
| Why is it different? | New mechanic or constraint | Just bigger numbers |
| Does the player need to adapt? | Yes, new strategy required | No, same strategy works |
| Does it feel fresh? | Yes, because decisions changed | No, it's just a reskin |

## 6. Activity Design Template

### Activity Structure
```
ACTIVITY: [Name]
TYPE: core / side / event / social / idle
SESSION LENGTH: [minutes]
FREQUENCY: [daily / weekly / event]
ENTRY CONDITION: [what the player needs to access it]
CORE VERB: [what the player does]
DECISION: [what choice the player makes]
REWARD: [what the player gets]
RISK: [what the player can lose]
EXIT CONDITION: [how the activity ends]
REPLAYABILITY ENGINE: [variance / mastery / content / social / economy / buildcraft]
```

## 7. Onboarding Beat Structure

### First 3 Minutes
- Emotional promise: Why does the player want to keep going?
- First action: What does the player DO immediately?
- First feedback: How does the game respond?

### First 10 Minutes
- First decision: Does the player make a real choice?
- First reward: Does the player get something meaningful?
- First loss: Does the player experience failure (and learn from it)?

### First 30 Minutes
- Core loop understanding: Can the player explain what they're doing?
- Next goal: Does the player know what to do next?
- Return hook: Why will the player come back tomorrow?

### Onboarding Red Flags
- Tutorial teaches menus before motivation
- Early rewards arrive before the player understands what they achieved
- Player is blocked by modal instructions for too long
- Multiple economies, upgrades, or meta systems unlock at once
- First session has no meaningful choice
